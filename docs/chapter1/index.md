## 前置知识

进入课程二之前，建议理解以下内容：

- Android 应用基本结构、生命周期和 Gradle 构建。
- Kotlin 基础语法、协程和常见集合操作。
- Web 运行环境、浏览器渲染、JavaScript 与 CSS 基础。
- 跨平台框架常见渲染模式：原生控件渲染、自绘渲染、WebView 渲染。
- 跨语言通信和跨运行时通信的基本成本。
- 原生能力调用中的权限、生命周期、线程和回调问题。

## 与课程二的关系

课程一回答“跨平台为什么能工作”，课程二回答“跨平台如何在真实工程里落地”。

在课程二中，所有技术栈都围绕同一个贯穿项目展开：

> 跨平台智能座舱电子手册与车辆助手

学习重点会从原理解释转向工程实现、性能评测、存量接入和企业选型。

## 复习清单

开始课程二之前，可以用以下问题自查：

- WebView、React Native、Flutter 和 Compose 的渲染路径有什么差异？
- JavaScript Bridge、Platform Channel、Native Module 的调用边界分别在哪里？
- 为什么跨平台框架的启动速度、包体积和内存占用会不同？
- 为什么同一个 UI 在不同技术栈里会有不同的布局和调试成本？
- 什么场景适合共享 UI，什么场景更适合只共享业务逻辑？
